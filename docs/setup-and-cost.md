# Setup And Cost

ForkFirst is designed to be useful as a free public repo. A new user can clone it, run it locally, and try demo mode without creating paid accounts.

Minimum local setup:

```bash
npm install
npm run dev
```

Open `http://127.0.0.1:3000`.

## GitHub Token

`GITHUB_TOKEN` is optional, but recommended. ForkFirst supports two modes:

- A server default token in `.env.local`.
- A user-provided token pasted into the app's "Bring your own keys" panel.

Without a token, GitHub's public REST API is limited to lower unauthenticated usage. With a token, ForkFirst can make many more GitHub API calls and the app is less likely to hit rate limits during repeated searches.

Use a fine-grained personal access token with read-only public repository access. Do not grant write access.

Add it to `.env.local`:

```bash
GITHUB_TOKEN=github_pat_your_token_here
```

Restart the dev server after changing `.env.local`.

## OpenAI Key

`OPENAI_API_KEY` is optional. ForkFirst supports two modes:

- A server default key in `.env.local`.
- A user-provided key pasted into the app's "Bring your own keys" panel.

Without it, ForkFirst uses deterministic demo analysis. GitHub search is still real, but the verdict is generated by local scoring rules.

With it, ForkFirst can use the AI analyst provider for better verdicts, summaries, and gap analysis. The default OpenAI model is `gpt-4.1-nano` because it is low-cost and strong enough for concise ranking/explanation work.

Add it to `.env.local`:

```bash
OPENAI_API_KEY=sk-proj_your_key_here
OPENAI_MODEL=gpt-4.1-nano
```

Restart the dev server after changing `.env.local`.

## Bring Your Own Keys

For an open-source deployment, prefer BYOK.

Users paste their GitHub token and AI provider key in the app. OpenAI, Groq, DeepSeek, and custom OpenAI-compatible providers are supported. Keys are session-only by default; persistent browser localStorage is opt-in. Keys are sent to the running ForkFirst server only for requests the user triggers. They are not saved in SQLite.

If you deploy a public hosted version, make this behavior clear in the README and UI. Do not log request bodies, because request bodies may contain user-provided keys.

## Local Data

ForkFirst stores local research data in `.forkfirst/forkfirst.sqlite`.

For a clean demo or test pass:

```bash
Remove-Item -Recurse -Force .forkfirst
```

Do not commit `.forkfirst/`, `.env.local`, `.next/`, or `node_modules/`.

## Vercel Cost

If you already have Vercel, the app itself can likely run inside your existing included usage while traffic is small.

The separate costs to watch are:

- GitHub API: no direct cost for normal public API usage, but rate-limited.
- OpenAI API: pay-per-use when `OPENAI_API_KEY` is enabled.
- Database: V1 currently uses local SQLite, which is good for local development but not ideal for serverless production persistence.

For a deployed Vercel version, replace local SQLite with Vercel Postgres, Neon, Supabase, Turso, or another hosted database. Otherwise saved research may not persist reliably across serverless instances.

Expected early usage:

- Local/private development: usually near zero infra cost.
- Light deployed usage on an existing Vercel account: likely within included Vercel usage.
- OpenAI usage: usually cents to a few dollars during testing, depending on how many idea checks and how much repository text is analyzed.

## Public Launch Notes

- Keep `.env.example` safe and placeholder-only.
- Prefer browser BYOK over shipping maintainer-owned production keys.
- Confirm no screenshots or docs expose private tokens.
- Run `npm test`, `npm run lint`, `npm run typecheck`, and `npm run build` before tagging a release.
